1.2 common.2-breakpoints Breakpoints

The project uses a mobile first approach. For responsive design and theming components uses several breakpoints for different screen resolutions. Each breakpoint is specified by the corresponding variable in pixels. To set breakpoints for particular block or element, mixins are used.


.class {
  ... // Set common styles and styles for "mobile" resolution

  @include tablet {
    ... // Set styles for "tablet" resolution
  }

  @include desktop {
    ... // Set styles for "desktop" resolution.
  }

  @include wide {
    ... // Set styles for "wide desktop" resolution.
  }

  @include x-wide {
    ... // Set styles for "extra wide desktop" resolution.
  }
}
Parameters:
  • $tablet-width
    Maximal width for "mobile" resolution and Minimal width for "tablet" resolution.
  • $desktop-width
    Maximal width for "tablet" resolution and Minimal width for "desktop" resolution.
  • $wide-desktop-width
    Maximal width for "desktop" resolution and Minimal width for "wide desktop" resolution.
  • $x-wide-desktop-width
    Maximal width for "wide desktop" resolution and Minimal width for "extra wide desktop" resolution.
